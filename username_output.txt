       SSH keys are a matching set of cryptographic keys which can be used for authentication. Each set contains a public and a private key.
       In a user key set, the private key remains on the system being used to access the remote system  and is used to decrypt information that is exchanged in the SSH protocol.
       A public key is used to encrypt information, can be shared, and is used by the user and the remote server. 
       How ssh authentication works?
       a. Client (has private key) initiate SSH connection
       b. Server (has public key) send random challenge message 
       c. Client response encrypt message with private key
       d. Server decrypt message with public key. If message match, client is authenticated
